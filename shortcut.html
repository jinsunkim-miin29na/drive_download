Shortcut setup

단축어 만들기

아이폰 단축어 앱에서 아래 순서대로 만들면 웹페이지의 `단축어 실행` 버튼이 실제 사진/동영상 저장 흐름으로 연결됩니다.

단축어 이름

Drive Album Save

액션 순서

  1. 단축어 앱을 열고 새 단축어를 만듭니다.
  2. 단축어 이름을 `Drive Album Save`로 바꿉니다.
  3. `입력에서 사전 가져오기` 액션을 추가하고 입력은 `단축어 입력`으로 둡니다.
  4. `사전 값 가져오기`로 `albumName`을 꺼내 변수 `Album Name`에 저장합니다.
  5. `사전 값 가져오기`로 `files`를 꺼내 변수 `Files`에 저장합니다.
  6. `사전 값 가져오기`로 `expectedCount`를 꺼내 변수 `Expected Count`에 저장합니다.
  7. `사전 값 가져오기`로 `callbackUrl`을 꺼내 변수 `Callback URL`에 저장합니다.
  8. `숫자` 액션을 추가하고 값을 `0`으로 둔 뒤 변수 `Saved Count`에 저장합니다.
  9. `반복` 액션을 추가하고 대상은 `Files`로 둡니다.
  10. 반복 안에서 `사전 값 가져오기`로 `downloadUrl`을 꺼냅니다.
  11. `URL 콘텐츠 가져오기` 액션으로 `downloadUrl`을 다운로드합니다.
  12. `사진 앨범에 저장` 액션으로 다운로드 결과를 `Album Name` 앨범에 저장합니다.
  13. 저장이 끝나면 `계산` 또는 `숫자` 액션으로 `Saved Count + 1`을 만들어 `Saved Count`에 다시 저장합니다.
  14. 반복이 끝난 뒤 `텍스트 바꾸기`로 `Callback URL` 안의 `{SAVED_COUNT}`를 `Saved Count`로 바꿉니다.
  15. `URL 열기` 액션으로 바꾼 콜백 주소를 열어 비교 결과를 확인합니다.

복사할 키 이름

아이폰에서 안 될 때

`사진 앨범에 저장` 액션에서 앨범 이름 변수를 못 받는 iOS 버전이면, 단축어 안에서 앨범을 직접 선택해야 합니다. 이 경우 웹페이지의 앨범 이름은 비교 표시용으로만 쓰입니다.

Google Drive 다운로드가 권한 오류로 실패하면 공유 폴더가 공개가 아니거나 API key 방식으로 접근할 수 없는 폴더입니다.
